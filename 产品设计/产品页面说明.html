产品概述

先对齐"做什么、为谁做、价值在哪"。

《记事卡》是一个"把一件事说清楚、找对人来帮忙"的互助协作工具。用户把待办/需求写成一张"记事卡"，通过一度人脉（好友）或直接邀请，把卡推给能帮上忙的人；好友还能把卡分享给自己的二度人脉，形成"互助网络"。核心不是聊天，而是让一件事和能解决它的人高效连接。

🎯 目标用户
有零散协作需求的人——帮朋友找资源、约人、询问信息、协调事务；不想拉群、不想在聊天里反复翻。重视"关系信任"与"轻量记录"。
💡 一句话价值
"把聊天里说的事，变成一张可协作、可追溯、能找对人帮的记事卡。"
🧩 产品定位
介于"待办清单"与"熟人协作网络"之间：比待办多了人脉协作，比社群轻了建群负担。
🔑 关键能力
记事卡创建、协助者邀请、二度人脉扩散、从聊天一键整理（A+C）、订阅消息提醒。

从聊天整理（取数方案）

A（微信 AI）为主 + C（剪贴板）兜底
研究结论 · 推荐方案：A（微信 AI）为主 + C（剪贴板）兜底 已核实

交互页面设计说明 · 从聊天整理 A 为主 + C 兜底
一、产品说明（目标与决策）
  • 目标：让用户从微信聊天快速生成记事卡，不在小程序内从头填写，可勾选/补改。
  • 主路径 A（推荐）：微信聊天选消息 → 转发给元宝/微信AI → 自动结构化 → 唤起记事卡预填。依赖微信 AI（方案第 9 节已规划）。
  • 兜底 C：复制聊天 → 打开记事卡 → 剪贴板自动识别 → 预览。零依赖、始终可用。
  • 图片补充 ②：wx.chooseMessageFile 直接从会话选截图/文件。
  • 已否决：转发给第三方小程序（无 API 接收聊天文本，不成立）。
二、交互流程与状态
  • 入口：事项页「从聊天整理」快捷入口 / 新建页「从聊天整理」。
  • 预览页：标题/描述/待确认/协助者候选 四块，默认全勾选，可取消或编辑。
  • 生成：进入 card-edit，字段已预填，可二次编辑后发布。
  • 边界：剪贴板为空 → 引导"去微信复制或选文件"；iOS 需用户手势触发读取。
  • 隐私：明确告知"不主动读消息"，内容仅经用户复制/选择后进入。

页面清单与核心流程

6 个主页面 + 3 条核心流程。
页面清单 6 主页面
① 事项（首页）
功能：我创建的 / 与我相关的记事卡列表；"从聊天整理"快捷入口；FAB 新建。
入口：底部 Tab 左。
状态：进行中 / 待确认 / 已完成（圆点+文字双编码）。
② 互助
功能：一度人脉横滑 → 点击刷新下方二度人脉卡片；空态引导。
入口：底部 Tab 右。
状态：二度只读 + "申请加入"按钮。
③ 我的主页
功能：个人资料 + 创建的 / 协助的 / 收藏 子标签 + 互助统计。
入口：事项页头像。
状态：统计数字（帮成 / 被帮 / 信任分）。
④ 记事卡详情
功能：详情 + 协助者 + 待确认 + 进度；创作者可编辑、可邀请、可审批。
入口：列表点击。
状态：创作者 / 一度协助者 / 二度只读 三种视图。
⑤ 新建记事卡
功能：标题/描述/待确认/协助者/可见性 线性分组；底部"从聊天整理"。
入口：FAB / 从聊天整理。
状态：草稿 → 发布。
⑥ 邀请好友
功能：底部 sheet 多选好友 + 群聊入口；复用申请加入弹窗视觉语言。
入口：详情页「邀请」。
状态：已选 / 未选；发送后订阅消息提醒。

03

数据模型与权限

4 个数据模型 + 一度/二度权限矩阵。
数据模型 4 个
👤 User
_openid / name / avatar / trustScore。每个用户唯一标识，关系与卡都以其为锚。
🔗 Relationship
from · to · degree(1|2) · source。一度=直接好友；二度=好友的好友。决定卡能否扩散。
📇 Card
_id · creatorId · title · desc · todos[] · helpers[] · isNetworkVisible · status · createdAt。记事卡本体。
🤝 JoinRequest
cardId · applicantId · degree · status(待审/通过/拒绝) · appliedAt。二度申请加入的凭证。
权限矩阵 一度 / 二度
操作创作者一度协助者二度访客
查看记事卡✓ 完全✓ 完全仅 isNetworkVisible=开
编辑内容✓部分（待确认）✗
邀请协助者✓✗✗
审批二度申请✓✗✗
申请加入——✓（触发审批）

isNetworkVisible（网络可见性开关）：控制该卡能否进入二度人脉扩散。关 → 仅一度可见；开 → 二度可浏览并可申请加入。新建页用"开关 + 一句人话说明"呈现，把字段翻译成用户能懂的取舍。

04

交互设计说明

让"既美观又易用"成立的关键决策。
🎯 符合习惯的框架
  • 沿用微信小程序顶栏 + 底部双 Tab（事项/互助），零学习成本。
  • 互助页"头像横滑→点击刷新下方列表"遵循熟人社交心智。
  • 二度只读 + 申请加入，权限与方案第 6 节一一对应。
✨ 状态编码与动效
  • 进行中/待确认/已完成均有文字 + 圆点双编码（不靠颜色单一维度）。
  • 状态切换用 ease-out 缓动，列表悬浮微抬升 2px，不回弹。
  • 弹窗从底部滑入，贴合小程序原生 sheet 习惯。
📥 空态 / 错误 / 边界
  • 互助页空态：引导"先添加协作过的人"。
  • 剪贴板为空 → 引导"去微信复制或选文件"。
  • iOS 需用户手势触发剪贴板读取（wx.getClipboardData）。
♿ 无障碍与一致性
  • 正文 14px 起，主按钮白字绿底对比 ≥ 4.5:1（WCAG AA）。
  • 触控目标 ≥ 44px。
  • 提供浅/深双主题，深色以"提亮表面"表达层级而非阴影。
🔒 隐私与信任
  • 明确告知"不主动读消息"，内容仅经用户复制/选择后进入。
  • 协助者候选来自聊天/@，可逐条删除，用户完全可控。
  • 订阅消息用于强提醒，不滥用推送。
🧱 与开发对齐
  • 设计 Token 已导出 design-tokens.wxss，rpx 直连小程序。
  • 页面路径：/pages/index · /pages/mutual · /pages/profile · /pages/card-detail · /pages/card-edit · /pages/invite · /pages/card-import。
  • 组件类与 HTML 一一对应，落地无歧义。
