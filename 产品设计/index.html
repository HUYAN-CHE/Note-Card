把"互助"做成有人情味的协作入口

以"纸感暖绿"为美学主张，用暖奶油纸张承载内容、以一抹清新绿作为协作信号色。界面遵循微信小程序的空间习惯，同时借由可信的视觉层级、教育性空状态与渐进式披露，让"看朋友需要什么、我能帮什么"变得自然。

目标用户
微信生态中以"我"为中心的协作网络用户——既有求助需求、也有助人意愿的个体（自由职业者、小商家、创作者、社群达人）。
核心任务
管理我的记事卡；浏览朋友/朋友的朋友需要什么；展示"我能提供的服务"；只读浏览他人记事卡并申请加入。
品牌调性
温暖、信任、协作、有人情味。不是冷冰冰的企业工具，而是"帮朋友"的暖色空间。

从聊天整理

A（微信AI）为主 + C（剪贴板）兜底
研究结论 · 推荐方案：A（微信 AI）为主 + C（剪贴板）兜底 已核实

硬约束：小程序无法主动读取聊天记录（微信隐私红线）。结合核实结论，推荐组合： A · 微信 AI 转发整理 为主路径（转发聊天给元宝/微信AI → 结构化 → 唤起记事卡；元宝 2026-05 已上线聊天总结，产品方案第 9 节规划微信AI接口）； C · 剪贴板粘贴 为始终可用的兜底（复制 → 打开记事卡 → wx.getClipboardData → 预览）。 两者都不要求用户在小程序内从头填写，AI 是主线的一部分（已规划），不是"后期才有的奢望"。

A · 微信 AI 转发整理 ✅ 推荐
聊天中选消息 → 转发给元宝/微信AI → 自动结构化 → 唤起记事卡预填。元宝已上线"聊天记录总结"，与方案第 9 节吻合。
C · 剪贴板粘贴（兜底）✅
复制聊天 → 打开记事卡 → wx.getClipboardData 自动识别 → 预览。零依赖、全平台、始终可用。
② 从聊天选图片/文件
在小程序内调 wx.chooseMessageFile 直接从会话选截图/文件，适合"聊天截图"类需求（图片补充）。

⚠️ 已移除"转发给第三方小程序"：普通 3P 小程序无 API 接收被转发的聊天文本，该路径不成立，已从方案删除。

交互页面设计说明 · 从聊天整理 A 为主 + C 兜底
一、产品说明（目标与决策）
  • 目标：让用户从微信聊天快速生成记事卡，不在小程序内从头填写，可勾选/补改。
  • 主路径 A（推荐）：微信聊天选消息 → 转发给元宝/微信AI → 自动结构化 → 唤起记事卡预填。依赖微信 AI（方案第 9 节已规划）。
  • 兜底 C：复制聊天 → 打开记事卡 → 剪贴板自动识别 → 预览。零依赖、始终可用。
  • 图片补充 ②：wx.chooseMessageFile 直接从会话选截图/文件。
  • 已否决：转发给第三方小程序（无 API 接收聊天文本，不成立）。
二、交互流程与状态
  • 入口：事项页「从聊天整理」快捷入口 / 新建页「从聊天整理」。
  • 预览页：标题/描述/待确认/协助者候选 四块，默认全勾选，可取消或编辑。
  • 生成：进入 card-edit，字段已预填，可二次编辑后发布。
  • 边界：剪贴板为空 → 引导"去微信复制或选文件"；iOS 需用户手势触发读取。
  • 隐私：明确告知"不主动读消息"，内容仅经用户复制/选择后进入。
03

交互与无障碍

让"既美观又易用"成立的关键决策。
🎯 符合习惯的框架
  • 沿用微信小程序顶栏 + 底部双 Tab（事项/互助），零学习成本。
  • 互助页"头像横滑→点击刷新下方列表"遵循熟人社交的心智模型。
  • 二度人脉只读 + 申请加入，权限与产品方案第 6 节一一对应。
✨ 渐进式披露与动效
  • 首屏只暴露"协作过的人 + 需要什么"，详情才展开协助者/待确认/进度。
  • 状态切换用 ease-out-expo 缓动，列表悬浮微抬升 2px，不回弹。
  • 弹窗从底部滑入，贴合小程序原生 sheet 习惯，降低中断感。
♿ 无障碍与一致性
  • 正文 14px 起、主按钮白字绿底对比 ≥ 4.5:1（WCAG AA）。
  • 状态不仅靠颜色：进行中/待确认/已完成均有文字 + 圆点双重编码。
  • 触控目标 ≥ 44px；提供浅/深双主题，深色以"提亮表面"表达层级而非阴影。
📝 新建 / 邀请闭环
  • 新建页用"标题→描述→待确认→协助者→可见性"的线性分组，减少一次性信息过载。
  • 网络可见性用开关 + 一句人话说明，把 isNetworkVisible 翻译成用户能懂的取舍。
  • 邀请复用底部 sheet 习惯，好友多选 + 群聊双入口，与申请加入弹窗保持同一视觉语言。
